Message Caching (Optional)

Create a new resource

  1. From the API Gateway Dashboard, select the price calculation API root element

  2. Click Actions , then select Create Resource

  3. Enter the following values

    • Resource name: medianPriceCalculator

  4. Click Create Resource

  5. Click Actions , then select Create Method.

  6. Select POST and select the check mark.

  7. For Post Setup, select the following values:

    • Integration Type: Lambda Function
    • Lambda region: (or region of your choice)us-east-1
    • Lambda function: (Select the function that appears.)serverless-immersion-day-MedianPriceCalculator-

  8. Click Save , then click Ok on the pop-up window.

Enable caching

In this example, we will create a query parameter to store the results based on the region attribute. Region can receive US or CA , United States or Canada values respectively.

  1. Click the POST method of the medianpriceCalculator API.
  2. From the API Console, click Method Request
  3. Expand URL Query String Parameter.
  4. Click Add Query String
  5. Enter the name region
  6. Click the check mark icon
  7. Check caching

Now that we are passing the region parameter as a query, we need to map that parameter to the built-in payload.

  1. Click medianpricecalculator and then click POST action .

  2. Click Integration Request from the API Console

  3. Extending mapping Template

  4. Click When no template is defined (recommended)

  5. Click Add Mapping Template

  6. Input for content type application/json

  7. Click the check mark icon

  8. Click on the application/json content type.

  9. Enter the following mapping for the “application/json” template

{
    "region": "$input.params("region")"
}
  1. Click Save

At this point, we’re basically taking the region value from the query parameter and mapping it to the request.

Deploy the API

Now it’s time to implement our new API.

  1. Click POST method of medianpricecalculator
  2. Click Action
  3. Click Deploy API
  4. Choose a dev stage
  5. Enter an option description Add new method with caching.
  6. Click Deploy

Now that the method is implemented, we can go ahead and enable caching.

  1. Click Stages under the CalculatePrice API

  2. Click dev stage

  3. Check Enable API Cache

  4. Select Cache: 0.5GB

  5. Enter Cache time-to-live (TTL): 60

  6. Leave all other fields as default

  7. Click Save Changes

  8. From the dev stage, click pricepermeter

  9. Click POST method

  10. Select Override for the method

  11. Uncheck Enable Method Cache (this option is unchecked by default)

  12. Click Save Changes Go ahead and create a new API using PostMan.

  13. Click Create a Request

  14. Enter the copied calling URL and append ?region=US to the end of the URL

  15. Change method type to POST from GET

  16. Click Body tab for Request

  17. Select the raw type in the radio buttons/drop-down

  18. Change the Content type to JSON

  19. Click Send
