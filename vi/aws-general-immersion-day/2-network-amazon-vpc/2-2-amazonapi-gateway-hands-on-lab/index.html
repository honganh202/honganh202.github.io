Amazon API Gateway bắt tay vào phòng thí nghiệm

Tổng quan về Cổng API của Amazon

Cổng API Amazon là một dịch vụ được quản lý hoàn toàn giúp các nhà phát triển dễ dàng tạo, xuất bản, duy trì, giám sát và bảo mật API ở mọi quy mô. API đóng vai trò là “cửa trước” để các ứng dụng truy cập dữ liệu, logic nghiệp vụ hoặc chức năng từ các dịch vụ phụ trợ của bạn. Sử dụng API Gateway, bạn có thể tạo API RESTful, API HTTP hoặc API WebSocket (cho phép các ứng dụng giao tiếp hai chiều theo thời gian thực). Cổng API hỗ trợ khối lượng công việc được chứa và không có máy chủ, cũng như các ứng dụng web.

API Gateway xử lý tất cả các tác vụ liên quan đến việc chấp nhận và xử lý tới hàng trăm nghìn lệnh gọi API đồng thời, bao gồm quản lý lưu lượng, hỗ trợ CORS, ủy quyền và kiểm soát truy cập, điều tiết, giám sát và quản lý phiên bản API. API Gateway không có phí tối thiểu hoặc chi phí khởi động. Bạn thanh toán cho các lệnh gọi API mà bạn nhận được và lượng dữ liệu được truyền đi, đồng thời với mô hình định giá theo bậc của Cổng API, bạn có thể giảm chi phí của mình khi quy mô sử dụng API của bạn tăng lên. Trong các phần chính của phòng thí nghiệm, bạn sẽ tạo một API gọi hàm lambda để tính giá của một ngôi nhà (giá mỗi foot vuông). Trong phần đầu tiên của phòng thí nghiệm, bạn sẽ triển khai một hàm lambda mẫu được xác định trước và tạo API back-end. Bạn sẽ kiểm tra các API của mình bằng cả bảng điều khiển Cổng API và Postman.

Trong các phần thực hành tùy chọn, bạn sẽ tìm hiểu cách lưu vào bộ nhớ đệm các phản hồi dịch vụ, điều tiết các yêu cầu và tạo các kế hoạch sử dụng khác nhau.

Kiến trúc tổng thể cho phòng thí nghiệm được mô tả ở trên.

Định cấu hình Cổng API của riêng bạn bằng cách đi qua các phòng thí nghiệm theo thứ tự bên dưới:

  • Điều kiện tiên quyết và triển khai Lambda
  • Tạo API đầu tiên của bạn

  • Xác thực yêu cầu
  • Xác thực và ủy quyền
